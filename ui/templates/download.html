Get Started with SMW Tracker

Connect your SNES to the cloud. Track splits, deaths, and PBs automatically. Share your runs live with the world.

What You Need
🐍
Python 3.10+
python.org/downloads — check "Add to PATH" during install
🔌
QUsb2Snes
Download QUsb2Snes — bridge between your hardware and the tracker
🎮
Compatible Hardware
SD2SNES / FXPak Pro, or a supported emulator (RetroArch, Snes9x with lua)
👤
SMW Tracker Account
Sign up here (free) — you'll need your API key from the Account page
Installation
1
Clone the repository
git clone https://github.com/zGuhT/smw-tracker.git
cd smw-tracker
2
Install dependencies
pip install -r requirements.txt
3
Start QUsb2Snes

Launch QUsb2Snes and make sure your SD2SNES/FXPak is connected, or your emulator has the connector running.

4
Run the tracker
# Find your API key at smwtracker.com → Account
python run_tracker.py --cloud --api-key YOUR_API_KEY
5
Set up your game

Load your ROM. The tracker auto-detects the game. Go to smwtracker.com → Games → [Your Game] → Setup to configure levels. Use the Capture button on each level while you're in that level to map SNES level IDs.

How It Works
⚡
Reads SNES memory 4× per second

Via QUsb2Snes — detects level, position, lives, animation state, game mode transitions

💀
Detects deaths automatically

Three methods: life counter, death animation (0x09), and game mode death sequence — works with kaizo 0-life hacks

🏁
Records splits on level exit

Goal tape, keyhole/secret exits, level-end modes, and level transitions — normal, secret, and special exits

☁️
Syncs to cloud every 0.5s

Sessions, splits, deaths, and progress pushed to smwtracker.com — anyone can watch live

Troubleshooting
"Can't connect to QUsb2Snes"

Make sure QUsb2Snes is running and shows your device. The tracker connects to ws://localhost:23074 by default. Change with --qusb-url ws://host:port.

"No default run for [game]"

Normal on first launch. Go to the game's Setup page on smwtracker.com to create a run definition with levels and capture SNES level IDs.

"Cloud push rejected: invalid API key"

Check your API key on the Account page. Ensure no extra spaces or line breaks in the key.

Deaths aren't being counted

The tracker uses three detection methods (life counter, animation 0x09, game mode 0x12). If your hack bypasses all three, please report it.

Splits aren't recording

Ensure levels are configured on the Setup page with captured SNES level IDs. Only levels in your run definition trigger splits.
